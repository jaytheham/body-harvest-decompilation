## Instructions for Contributors

1. **Before Starting Work**:
   - Check this document to ensure the function isn't already claimed
   - Create a GitHub issue to claim the function
   - Update this document with your name in the "Contributor" column
   - Change status to "Claimed"

2. **When Starting Work**:
   - Change status to "In Progress"
   - Add a link to your GitHub issue in the "GitHub Issue" column

3. **When Completing Work**:
   - Change status to "Needs Review"
   - Submit a Pull Request
   - Link the PR to your GitHub issue

4. **After Merging**:
   - Change status to "Matched"
   - Add any notes about techniques used that might help other contributors

## Adding New Functions

When new functions are discovered or existing ones are split:
1. Add them to the appropriate section
2. Set initial status to "Available"
3. Include the function name and any known notes

## Notes Column Usage

Use the notes column for:
- Links to helpful resources or references
- Known challenges or complexities
- Dependencies on other functions
- Tips or tricks that might help
